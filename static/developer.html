Quick Start

Arbiter exposes an OpenAI-compatible API. Any SDK or tool that works with the OpenAI API works with Arbiter — just change the base URL.

Base URL: https://arbiter.chkoushik.com
Endpoint: /v1/chat/completions

Python (OpenAI SDK)

pythonfrom openai import OpenAI client = OpenAI( base_url="https://arbiter.chkoushik.com/v1", api_key="your-gateway-token", # from Settings → Gateway Tokens ) response = client.chat.completions.create( model="auto", # let Arbiter choose the best model messages=[{"role": "user", "content": "Hello!"}], ) print(response.choices[0].message.content)

cURL

bashcurl -X POST https://arbiter.chkoushik.com/v1/chat/completions \ -H "Authorization: Bearer your-gateway-token" \ -H "Content-Type: application/json" \ -d '{ "model": "auto", "messages": [{"role": "user", "content": "Hello!"}] }'

Model Selection

Model ValueBehavior
"auto"Arbiter classifies the request and picks the best free-tier model
"gemini-2.5-flash"Use a specific model — Arbiter routes to the correct provider
"nvidia/nemotron-3-super-120b-a12b"Fully-qualified model — routed directly to NVIDIA NIM
"cfworker/my-worker"Route to a deployed Cloudflare Worker
⚠️ Free Tier Mode: By default, Arbiter routes through free API providers (Gemini, Groq, NVIDIA, etc.). These have rate limits and may not be suitable for production workloads. Configure paid API keys in Settings for production stability.

Authentication

All /v1/* endpoints require authentication via Bearer token.

Gateway API Token

httpAuthorization: Bearer arbiter-sk-xxxxxxxxxxxxxx

Create tokens in Settings → Gateway Tokens. Each token has:

  • Unique name for identification
  • Optional expiry date
  • Enable/disable toggle
  • Per-token analytics (requests, models used, providers hit)
  • Routing policy — restrict which models this token can use

Routing Policies (per token)

PolicyBehavior
autoArbiter chooses freely from all available models (default)
restrictedOnly models in the token's allowed_models list are used. Order = priority.
preferredPreferred models tried first, falls back to full auto if exhausted

Routing Algorithm

Arbiter uses a multi-level intelligent routing system:

1. Cache Check

For requests with temperature ≤ 0.3, check Redis cache first. Instant response on hit.

2. Candidate Chain

Build an ordered list of (provider, model) pairs based on:

  • Explicit model name → route to owning provider
  • Token count → prefer providers with sufficient context window
  • Capability tags → code, reasoning, creative → different models
  • Gateway routing policy → filter/reorder based on token config

3. Gateway Policy Filter

If the calling token has routing_policy: "restricted", only allowed models survive. If "preferred", preferred models are moved to the front of the chain.

4. Key Rotation (per model)

For each candidate, rotate through all API keys using weighted scoring:

formulascore = (rpm_available × 0.25) + (tpm_available × 0.15) + (daily_available × 0.40) + (health × 0.20)

5. Fallback

On rate limit → try next key. On provider error → try next model. All exhausted → 503.

Provider Priority (default)

textgemini → groq → cerebras → nvidia → zai → cloudflare → openrouter → cohere → huggingface → pollinations → ollama → routeway → lightning

Gateway Token Management

Gateway tokens control access to Arbiter's completions API. Each token can have its own routing policy.

Create a Token with Model Restrictions

bash# Create a token restricted to specific models curl -X POST https://arbiter.chkoushik.com/api/gateway/tokens \ -H "Cookie: arbiter_session=..." \ -H "Content-Type: application/json" \ -d '{ "name": "OpenClaw Production", "routing_policy": "restricted", "allowed_models": [ "nvidia/nemotron-3-super-120b-a12b", "mistralai/mistral-medium-3.5-128b", "meta/llama-3.3-70b-instruct" ] }'

Update Routing Policy

bash# Switch to preferred mode (try these first, fall back to auto) curl -X PATCH https://arbiter.chkoushik.com/api/gateway/tokens/gwtk_abc123 \ -H "Cookie: arbiter_session=..." \ -H "Content-Type: application/json" \ -d '{ "routing_policy": "preferred", "allowed_models": ["nvidia/nemotron-3-super-120b-a12b"] }'

Block Specific Models

json{ "blocked_models": ["gemini-2.5-pro", "gpt-4"] }

Blocked models are never used regardless of routing policy. Useful for preventing accidental use of expensive paid models.
