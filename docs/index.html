One endpoint for every free-tier LLM.

Arbiter is a self-hosted, production-ready gateway that aggregates 11+ LLM providers behind a single OpenAI-compatible API. It routes each request to the best model using complexity analysis, rotates keys across accounts, falls back automatically, caches responses, and gives you a real-time observability dashboard — so you maximise free quota and never hit a wall.

Capabilities

Everything a gateway should do — and more

Built for multi-agent workloads that burst concurrent requests. Arbiter squeezes the most out of free tiers while staying observable and resilient.

🧠

Complexity-aware routing

Every request is scored TRIVIAL→EXPERT. Hard prompts go to 120B–671B flagships; trivial ones to fast 7B models — matching capability to cost automatically.

⚖️

Weighted key rotation

Multiple accounts per provider, scored in real time by remaining RPM / TPM / daily / monthly quota. Load spreads to the key with the most headroom.

🔁

Two-level fallback

Key → model hierarchy → cross-vendor. A failure on one account or model transparently cascades to the next until the request succeeds.

🚦

Circuit breaker

A provider/model that fails repeatedly is skipped for 5 minutes — so a reliably-broken upstream never burns latency on every request. Rate-limits never trip it.

⚡

Predictive rate-limiting

Keys are skipped at ~85–95% of quota so the router routes around exhaustion before a 429 ever reaches you. Retry-after-aware cooldowns.

🗜️

Redis response cache

Deterministic requests (temp ≤ 0.3) are cached & gzip-compressed. Cache key includes tools / response_format / seed so tool & JSON-mode calls never collide.

🌐

Real-time web search

Opt-in X-Arbiter-Realtime header grounds answers in live Tavily results with numbered citations + Gemini Google-Search grounding.

🛠️

Native tool calling

Pass an OpenAI tools array — Arbiter auto-routes to tool-capable providers and forwards function definitions verbatim.

📊

Real-time observability

Dashboards for per-provider health, per-key quota gauges, latency percentiles, token spend, anomaly detection, and a 180-day audit log.

🔐

Enterprise auth

Google SSO with admin approval for the UI; per-token Bearer auth for the API with named tokens, routing policies, model allow/block lists, and rate caps.

🎨

Free image generation

Built-in /v1/images/generations via Pollinations — no key required — plus a UI gallery.

🧩

Bring your own provider

Add any OpenAI- or Anthropic-compatible endpoint from the UI with a DNS-aware SSRF guard. Live model discovery included.

Architecture

How a request flows

From your OpenAI SDK call to the chosen upstream model — with caching, scoring, and fallback at every hop.

STEP 1

Authenticate

Bearer gateway-token check (constant-time) + per-token sliding-window rate limit.

STEP 2

Cache & classify

Redis cache lookup, then complexity + intent classification of the prompt.

STEP 3

Build chain

Candidate (provider, model) list ordered by capability, health & quota — circuit-broken pairs skipped.

STEP 4

Score & route

Best key chosen by weighted availability; request sent over a pooled HTTP client.

STEP 5

Fallback or return

On failure, rotate key → next model → next vendor. On success, cache & record stats.

⚙️ Stack: Python 3.12 · FastAPI · Redis 7 · Docker Compose. Two containers (gateway + redis), one .env, port 8080. Background schedulers handle backups (S3), weekly model-health probes, daily email reports, and 180-day log retention.

Quick start

One command to clone, one to configure, one to launch. Docker does the rest.

🐳 Docker (recommended)
⚙️ Configure .env
✅ Verify
# 1. Clone
git clone https://github.com/koushikch7/arbiter.git && cd arbiter

# 2. Configure (add at least one provider key)
cp .env.example .env
nano .env

# 3. Launch (gateway + redis)
docker compose up -d --build

# Open the dashboard
open http://localhost:8080/dashboard
# Minimum: one provider key + a gateway token.
# Add one or more keys per provider, comma-separated.

GEMINI_API_KEYS=AIzaSy...your-key
GROQ_API_KEYS=gsk_...your-key
CEREBRAS_API_KEYS=csk-...your-key

# Protect the API (comma-separated list of valid Bearer tokens)
GATEWAY_API_KEYS=gwtk_choose_a_strong_token

# Optional: Google SSO for the admin UI, SMTP, S3 backups…
# See .env.example for the full list.
# Health check
curl http://localhost:8080/health
# → {"status":"ok","providers":[...],"version":"1.21.0"}

# First completion
curl http://localhost:8080/v1/chat/completions \
  -H "Authorization: Bearer gwtk_your_token" \
  -H "Content-Type: application/json" \
  -d '{"model":"auto","messages":[{"role":"user","content":"Hello!"}]}'

API endpoints

A drop-in OpenAI surface plus Arbiter extensions. Authenticate every /v1/* call with Authorization: Bearer <gateway-token>.

Core OpenAI-compatible API

MethodEndpointDescription
POST/v1/chat/completionsChat completion (streaming & non-streaming). Smart auto-routing.
GET/v1/modelsList all available models across providers.
POST/v1/images/generationsGenerate images (free via Pollinations).
GET/v1/images/modelsList image-generation models & sizes.
GET/healthHealth + provider list + version (no auth).

Arbiter request controls

HeaderBody fieldEffect
X-Arbiter-Prioritymetadata.priorityspeed | quality | balanced — scoring bias
X-Arbiter-Prefer-Providermetadata.prefer_providerBoost a provider in auto routing
X-Arbiter-Fallbackfallbacknone | same_provider | chain
X-Arbiter-Realtimemetadata.realtimetrue → Tavily web search + OpenRouter :online
?vendor=  /  ?force_model= (query)Pin a provider or exact model, bypassing auto-routing

Operating Arbiter

User guide highlights

Manage everything from the web UI — no restarts, no YAML surgery.

🎛️ Admin dashboards

  • /dashboard — live provider & per-key health, success rates, quota bars
  • /analytics — latency percentiles, token spend, anomaly detection, CSV/JSON export
  • /settings — add/remove/test keys, toggle providers & models, routing policy, cache
  • /playground — chat UI with model picker & smart-route option
  • /logs & /logs/persistent — live buffer + 180-day audit viewer
  • /backup — S3 backup & restore management

🔑 Runtime management

  • Add & rotate provider keys at runtime — written to .env, auto-tested
  • Issue named gateway tokens with per-token rate limits & model allow/block lists
  • Tag keys #paid to reserve frontier models for billing-enabled accounts
  • Per-token routing policies: auto · restricted · preferred
  • Google SSO with admin-approval workflow for UI access
  • Weekly AI-analysis email reports & major-change dashboard banners
