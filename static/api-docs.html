Arbiter

An intelligent, OpenAI-compatible API gateway that routes requests across multiple free-tier LLM providers with automatic key rotation, rate-limit tracking, and response caching.

Base URL
http://localhost:8000
API Version
v1.0.0
Protocol
OpenAI-compatible REST

Quick Start

curl -X POST http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-gateway-key" \
  -d '{
    "model": "gemini-2.5-flash-lite",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'

Features

  • ✓ OpenAI-compatible /v1/chat/completions endpoint
  • ✓ 8 providers: Gemini, Groq, Cerebras, Cloudflare, OpenRouter, Cohere, HuggingFace, Pollinations
  • ✓ Intelligent routing based on model name, token count, and content type
  • ✓ Automatic key rotation with weighted scoring across multiple accounts
  • ✓ Response caching for deterministic requests (temperature ≤ 0.3)
  • ✓ Vendor/model override via query parameters
  • ✓ Optional Cloudflare Access JWT authentication
  • ✓ Cloudflare Workers AI management API

Authentication

Bearer Token Auth

When GATEWAY_API_KEYS is set, all protected endpoints require a Authorization: Bearer <key> header. If the environment variable is empty, authentication is disabled. 

Authorization: Bearer your-gateway-key

Multiple keys can be provided as a comma-separated list in GATEWAY_API_KEYS.

Exempt Paths (No Auth Required)

GET/health
GET/docs
GET/redoc
GET/openapi.json
GET/api-docs
GET/dashboard
GET/dashboard/stats
GET/v1/models

Error Response

When auth fails, the gateway returns HTTP 401:

{
  "error": {
    "message": "Invalid API key",
    "type": "authentication_error",
    "code": 401
  }
}

Endpoints

POST /v1/chat/completions

Create a chat completion. OpenAI-compatible request/response format.

Query Parameters
vendor Force a specific provider: gemini, groq, cerebras, cloudflare, openrouter, cohere, huggingface, pollinations
force_model Override the model ID (e.g. llama3.1-8b). Overrides the body model field.
Request Body
{
  "model": "gemini-2.5-flash-lite",
  "messages": [
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "Hello!"}
  ],
  "temperature": 0.7,
  "max_tokens": 512,
  "top_p": 1.0
}
Response
{
  "id": "chatcmpl-abc12345",
  "object": "chat.completion",
  "created": 1711234567,
  "model": "gemini-2.5-flash-lite",
  "choices": [{
    "index": 0,
    "message": {"role": "assistant", "content": "Hello! How can I help you today?"},
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens": 24,
    "completion_tokens": 12,
    "total_tokens": 36
  }
}
GET /v1/models

List all available models across all configured providers.

{
  "object": "list",
  "data": [
    {"id": "gemini-2.5-flash-lite", "object": "model", "owned_by": "gemini"},
    {"id": "llama-3.1-8b-instant",  "object": "model", "owned_by": "groq"},
    ...
  ]
}
GET /health

Gateway health check. Always accessible without auth.

{
  "status": "ok",
  "redis": "connected",
  "providers": ["gemini", "groq", "cerebras"],
  "version": "1.0.0"
}

Cloudflare Workers AI Management

GET /cloudflare/models

List available Workers AI text-generation models.

POST /cloudflare/workers

Create a new AI Worker with a specified model binding.

{"name": "my-ai-worker", "model": "@cf/meta/llama-3.1-8b-instruct", "description": "My AI worker"}
GET /cloudflare/workers

List all existing workers in the account.

DELETE /cloudflare/workers/{script_name}

Delete a specific worker script.

Model Routing Priority

  1. Gemini – largest context, highest quality
  2. Groq – fastest, highest RPM
  3. Cerebras – 1M tokens/day free
  4. Cloudflare Workers AI – 300 RPM free
  5. OpenRouter – many free models
  6. Cohere – 256K context
  7. HuggingFace – open model selection
  8. Pollinations – fully anonymous, no key

Smart Routing Rules

  • @cf/ in model → Cloudflare
  • gemini in model → Gemini
  • llama3.1-8b in model → Cerebras
  • command-r in model → Cohere
  • zephyr / qwen/qwen in model → HuggingFace
  • Token count > 100K → Gemini first
  • Token count < 4K → Groq first (fastest)
  • Code keywords detected → Gemini + Groq priority
